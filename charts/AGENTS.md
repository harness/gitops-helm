# AGENTS.md — charts/

The actual Helm chart. ~95% of repo activity happens here.

## Purpose

Helm chart `gitops-helm` (see `Chart.yaml`) that installs the Harness GitOps Agent plus an embedded Argo CD via the upstream `argo-cd` sub-chart from argoproj/argo-helm.

## Key Files

- `Chart.yaml` — name, `version:` (bump on user-facing changes), dependencies. SOURCE OF TRUTH for chart version.
- `Chart.lock` — generated by `helm dependency update`. Commit it.
- `values.yaml` — default values + user-facing API. Every new value needs a default + docs.
- `templates/_common.tpl` — shared template helpers (names, labels, annotations).
- `templates/gitops-agent/` — gitops-agent Deployment, ConfigMap, RBAC, secrets.
- `templates/argocd-*/` — wrappers / overrides on top of the argo-cd sub-chart.
- `templates/redis/` — Redis (cluster-internal or external w/ TLS — CDS-125210).
- `templates/upgrader/` — agent self-upgrader.
- `charts/argo-cd-9.0.0.tgz` — vendored sub-chart tarball. Do not edit. Use `helm dependency update` to refresh.

## Working Here

```bash
# After editing Chart.yaml dependencies
helm dependency update .

# Lint
helm lint .

# Render with defaults
helm template gitops-helm .

# Render with overrides (preferred when changing template logic)
helm template gitops-helm . -f /path/to/test-values.yaml

# Pre-flight install
helm install gitops-helm . -n argocd --create-namespace --dry-run --debug
```

## Patterns

- **Conditional sub-chart**: argo-cd is included via `condition: argo-cd.enabled` in `Chart.yaml`. Templates in `argocd-*/` should respect this flag.
- **Existing secrets**: User-supplied secrets are referenced via `agent.existingSecrets.*` in `values.yaml` (token, CA bundle, mTLS, redis TLS). When adding a new secret-like input, follow this pattern — never force users to inline credentials.
- **Naming**: Use helpers in `_common.tpl` (`{{ include "gitops-helm.fullname" . }}` etc.) rather than hand-rolling names.
- **Image overrides**: All image refs go through `values.yaml` keys so customers can override `repository` / `tag` / `pullPolicy` / `imagePullSecrets`.

## DOs

- Bump `Chart.yaml` `version:` on every user-facing change in this directory.
- Update root `CHANGELOG.md` describing the change.
- Run `helm lint .` and `helm template .` before committing any template change.
- When changing values defaults, render with both old and new values to confirm nothing user-facing regressed.
- Test sub-chart bumps end-to-end on a devspace cluster — argo-helm has shipped breaking changes between minors.

## DON'Ts

- Don't edit `charts/argo-cd-*.tgz` — managed by `helm dependency update`.
- Don't edit `Chart.lock` by hand — let `helm dependency update` regenerate it.
- Don't add new dependencies to `Chart.yaml` without discussing — every dep balloons the install footprint and CVE surface.
- Don't hardcode customer-specific values in `values.yaml` — defaults must work for every deployment.
